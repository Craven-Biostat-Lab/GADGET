GADGET User Manual

GADGET (Genes Associated by Documents, Genes, Events, and Text) is a tool that allows users to search and navigate the biomedical literature that is associated with specific genes. GADGET can identify the genes and keyphrases that are strongly associated with arbitrary text queries, and it can also show interactions that describe automatically extracted relationships among the genes.

GADGET has three main views: The gene search sifts through literature to identify genes associated with a particular topic. The keyphrase search lets you see keyphrases associated with topics and genes in the biomedical literature. The interaction search sifts through interactions between genes that have been automatically extracted from biomedical abstracts.

The gene search

GADGET's gene search returns genes that are associated with the abstracts in PubMed that match a given query. When you supply GADGET with a query, GADGET will return a list of genes related to your query in the literature, ordered by the approximate "importance" of the query for each gene: if you search for "surfactant," genes for which "surfactant" is more strongly associated will appear higher in the list.

GADGET gene search queries can include keyword searches, lists of gene symbols, a species, and an option to include homologs. GADGET will find PubMed abstracts that match your keyword query and refer to all of the genes that you specify, and then return genes found in those abstracts.

There are several different scores that GADGET can use to the rank genes that satisfy a given query. (Read about them here.) To change which score GADGET uses to sort the genes, just click on the name of that score in the result table header.

You can see a list of abstracts for each gene that match your query by clicking on the "show abstracts" link for each gene in the gene search results.

You can download gene search results as a CSV or XML. When you download gene search results, the results will contain all of the genes that match your query, not just the genes displayed on the page. The XML representation uses the DTD found here.

To search for genes, keywords, or interactions, you can enter one or more key terms. GADGET searches for abstracts that contain your query terms, and then links them to genes or interactions. Query terms in GADGET are not case sensitive (although Boolean operators are).

You will often get better results by using more specific terms in your searches. Avoid using common terms like "gene" in your query; they don't usually add much value to the results, and they can slow down GADGET.

GADGET supports Boolean queries for more specific searches. Terms can be separated with AND and OR. An AND query will match abstracts that contain both terms, while an OR query will match abstracts that contain either term. Terms separated by spaces are automatically treated as an AND query. You can also exclude terms from your query by using NOT followed by the term. You can use parentheses to make complex Boolean queries.

Examples:

Match genes or interactions referenced in abstracts matching both "Cat" and "Dog":
Cat AND Dog or just Cat Dog
Match genes or interactions referenced in abstracts matching "Cat" or "Dog", but not "Fish":
Cat OR Dog NOT Fish
Match genes or interactions referenced in abstracts matching "Cat", or both "Dog" and "Fish":
Cat OR (Dog AND Fish)

To do an exact match on a multi-word phrase, enclose the phrase in quotes.

Match abstracts containing the terms "Embryonic", "Stem", and "Cell":
Embryonic stem cell
Match abstracts containing "Embryonic stem cell" all together:
"Embryonic stem cell"

You can also restrict abstracts by the year they were published by entering year: followed by the year. You can specify ranges of years by using year:[startyear to endyear]. You can leave either startyear or endyear blank for an open-ended range.

Match abstracts published in 2011:
year:2011
Match abstracts containing "apoptosis" published between 2005 and 2009 (inclusive):
apoptosis year:[2005 to 2009]
Match abstracts containing "surfactant" published in or before 2007:
surfactant year:[to 2007]
Match abstracts containing "chitin" published in or after 2009:
chitin year:[2009 to]

GADGET accepts wildcards to match inexact terms. Including a question mark (?) in a term will match any single character, while an asterisk (*) will match any number of arbitrary characters. Note that using a wildcard character at the beginning of a term will significantly slow GADGET down. You cannot use wildcard characters within quotation marks.

Match abstracts containing "text", "test", etc.:
te?t
Match abstracts containing "BLAST", "blastula", "blastema", etc.:
blast*

GADGET uses the Whoosh query language for matching abstracts. Not all features of the query language are applicable to GADGET.

You can enter a list of genes to search for related genes and keyphrases. GADGET will find abstracts that mention your supplied genes, and show you genes or keyphrases from those abstracts.

The interaction search doesn't support all of the gene entry features supported by the other search modes. See searching interactions by gene symbol.

You can enter genes using their Entrez Gene symbols or Gene ID's (eg. TAL1 or 6886.)

If you provide GADGET with a list of genes, you can select whether GADGET should use abstracts that mention any (at least one) or all of your genes.

You can use boolean operators (AND, OR, and NOT in caps) in your gene list. These operators override your any / all choice.

You can also use * for wildcard matches.
With the any option selected, match abstracts that mention either SHH or LBH (or both.) With the all option selected, match abstracts that mention both SHH and LBH.
SHH LBH
The same as the previous example, using Entrez Gene ID's instead of gene symbols:
6469 81606
Match abstracts that mention PECAM1 and either KDR or TIE1 (or both):
PECAM1 AND (KDR OR TIE1)
Match abstracts that don't mention TWIST1:
NOT TWIST1
Match all of the gene symbols that start with HOX
HOX*

You can also upload a file with a list of genes. The file needs to have one gene symbol or Entrez Gene ID per line.

Back to top Back to GADGET Gene search scores

In the gene search, GADGET can use different scores to determine which genes are most important to your query.

When you do a search, GADGET finds all of the abstracts that match your query, and then finds genes that are mentioned in those abstracts. GADGET computes scores for each gene based upon: the number of abstracts matching the query for each gene, the total number of abstracts associated with each gene, and the total number of abstracts matching the query.

Each of the available scores takes some of these factors into account in different ways. You can rank the genes by a score by clicking on the score's name in the table header.

F1 scoreThe F1 score is a balanced measurement of precision and recall. It takes both the proportion of matching abstracts for each gene and the total number of matching abstracts into account. The F1 score represents a tradeoff between the "Adjusted precision" and "Matching abstracts" scores. This score is used by default.
Adjusted precisionThe adjusted precision score is based upon the proportion of a gene's abstracts that match your query. For each gene in your query, adjusted precision is calculated as (number of abstracts for the gene matching the query) / (10 + total number of abstracts for the gene). The precision is adjusted by adding a "pseudocount" of 10 to the denominator, so that a gene with 10/10 matching abstracts scores higher than a gene with 1/1 matching abstracts.
Matching abstracts"Matching abstracts" is the number of abstracts for the gene that match your query. This score is useful for finding genes that have larger amounts of published literature relevant to your query, but genes with more total abstracts tend to be ranked higher even if they are less relevant to your query.
Total abstracts"Total abstracts" shows the total number of abstracts associated with each gene. This number does not change based upon your query.

GADGET also computes a p-value for each gene using the hypergeometric test. The null hypothesis for the test is that the fraction of abstracts that match the query for a given gene is equal to the "background" fraction of abstracts that match the query in the entire corpus of abstracts. The p-value thus indicates how confident we are that a gene's abstract set is enriched for the query.

When you do a GADGET gene search, all of the genes in your results will be from a particular species. GADGET allows you to select from several species to view genes for. The currently-supported species are:

NCBI Taxonomy IDName
9606Homo Sapiens (default)
10090Mus musculus
559292Saccharomyces cerevisiae

For gene search queries, when you provide a list of gene symbols, GADGET will match them to genes from your selected species.

GADGET's interaction search does not take a species as input. GADGET's interaction search currently uses only human genes (and homologs.)

Back to top Back to GADGET Optional inclusion of homologs

When finding and scoring genes that match your query, GADGET's gene search can optionally take homologs into account. To calculate scores for each gene, GADGET counts the number of abstracts matching your query that each gene appears in. If you choose to include homologs, GADGET will also count abstracts that refer to homologs of each gene.

The groups of homologs that GADGET uses come from Homologene. Homologs can be either from the same species as each gene, or from different species.

Back to top Back to GADGET Using the gene search without a browser

If you want to automatically access GADGET's gene search without using a web browser, you can download comma-delimited CSV or XML files by fetching http://gadget.biostat.org/gadget/genelist? and sending arguments via the query string at the end of the URL. To download data, the query string must include &download=CSV or &download=XML, depending on which format you want.

To send information to GADGET, just append it to the url. Aguments are given by the name of the argument, an equals sign (=), and the argument's value, e.g. argument=value. Multiple arguments should be separated by ampersands (&). Here's a list of the arguments you can send:

downloadThis argument is required to download the results as a CSV or XML file. To download a CSV, use &download=CSV. To get XML, use &download=XML. The XML representation uses a DTD file that can be found here.
qThe keyword query terms of your search.

In most browsers, you can include spaces and quotation marks in the URL. Special characters, like "&" and "/", must be percent encoded.

Either q or genes (or both) is required for GADGET gene search queries.
genesA list of gene symbols and/or Entrez Gene IDs to search for. You can use the same format and all of the same features for entering genes as you can in the graphical interface.

If you provide gene symbols, GADGET will match them to genes for your selected species

Either q or genes (or both) is required for GADGET gene search queries.
geneopThe "gene operator" to use for your list of genes - whether to match abstracts that mention any or all of your genes (see gene entry for the gene search and keyphrase search.) If geneop isn't provided, any will be used by default.
speciesThe species to use for your gene search. All of the genes in your gene search results will belong to this species.

To specify a species, use the species's NCBI taxonomy ID, (e.g. &species=9606 for humans). See here for a list of currently available species and taxonomy IDs. If you don't specify a species, homo sapiens will be used by default.
usehomologsWhether or not to include homologs when calculating scores for each gene. (See here.) To include homologs, include &usehomologs=1 in your query string. If usehomologs is omitted, homologs will not be included by default.
orderbyWhich score to use to rank the results.

There are 4 possible values for orderby. The possible values are f1_score, which will order the genes by the F1 score; adjusted_precision, which will order the genes by the adjusted precision; matching_abstracts, which will order the genes by the number of abstracts matching your query; and total_abstracts, which will order the genes by the total number of abstracts for each gene.

If the orderby argument is omitted or invalid, GADGET will default to the F1 score. See above for more information about the different scores.
limitThe maximum number of genes to return. This argument should be a number. If limit is not included or is invalid, GADGET will return all of the genes found in the search.
offsetThe rank (minus one) of the first gene to include in the results. To omit the first 100 genes, include &offset=100. If offset is omitted or invalid, GADGET will not omit any genes.

If no abstracts or genes match your query, or if q is omitted or invalid, the GADGET server will issue a 404 error instead of returning a CSV file.

Examples:

Download a CSV of genes matching "arteriole", using homologs:
http://gadget.biostat.wisc.edu/gadget/genelist?download=CSV&usehomologs=1&q=arteriole
Download a CSV of mouse genes matching "mesoderm", ranked by adjusted precision:
http://gadget.biostat.wisc.edu/gadget/genelist?download=CSV&q=mesoderm&orderby=adjusted_precision
Download XML of the top 150 genes for mice matching "Loop of Henle", using homologs:
http://gadget.biostat.wisc.edu/gadget/genelist?download=XML&species=10090&q=Loop of Henle&limit=150&usehomologs=1
Download XML of the top 80 genes related to CFTR matching "cystic fibrosis", ranked by number of matching abstracts:
http://gadget.biostat.wisc.edu/gadget/genelist?download=XML&q=cystic fibrosis&genes=CFTR&limit=80&orderby=matching_abstracts
Back to top Back to GADGET The keyphrase search

GADGET's keyphrase search shows you a list of keyphrases that are associated with a certian set of the biomedical literature. You can search for keyphrases by entering keywords, a list of genes, a species, and an option to include homologs. GADGET's keyphrase search supports all of the same queries as the gene search.

When you do a GADGET keyphrase search, GADGET finds all of the abstracts that match your query, and finds the 1-word, 2-word, and 3-word keyphrases that are contained in the abstracts. It computes a few scores for each keyphrase, and sorts them to show you the keyphrases that are most relevant to your query.

The ends of many of the keyphrases in the results are cut off, because GADGET uses "stemming" to group related words together.

You can see a list of abstracts that contain each keyphrase, as well as which of your supplied genes match each keyphrase, by clicking on the "show abstracts" links in the search results.

You can download keyphrase search results as a CSV or XML. When you download keyphrase search results, the results will contain all of the keyphrases that match your query, not just the keyphrases displayed on the page.

Back to top Back to GADGET Keyphrase search scores

GADGET's keyphrase search provides several different scores for each keyphrase in your search results, that you can use to sort the keyphrases and determine which are most important to your query.

When you do a GADGET keyphrase search, GADGET finds all of the abstracts that match your query, and then collects, scores, and sorts all of the keyphrases contained in the abstracts. There are two families of scores: some are based on the number of abstracts that match each keyphrase, and some are based on the number of genes that match each keyphrase. The gene-based scores are only calculated when you supply gene symbols in your query.

The abstract f1 score. You can sort the keyphrases by different scores by clicking on their names at the top of the keyphrase search results.

Abstract F1 scoreThe F1 score is a balanced measurement of precision and recall. It takes both the proportion of matching abstracts for each keyphrase and the total number of matching abstracts into account. The F1 score represents a tradeoff between the "Abstract precision" and "Abstract recall" scores. This score is used by default.
Abstract precisionOf all of the abstracts associated with this keyphrase, what proportion of them match your query?
Abstract recallOf all of the abstracts that match your query, what proportion of them contain this keyphrase?
Matching abstractsThe number of abstracts containing this keyphrase that match your query.
Total abstractsThe number of abstracts containing this keyphrase in all of the literature that GADGET uses.
Gene F1 ScoreA balanced measurement of "gene precision" and "gene recall."
Gene precisionOf all of the genes associated with this keyphrase, what proportion of them are in the list of genes that you supplied? (A gene is "associated" with a keyphrase if it is mentioned in an abstract that contains that keyphrase.)
Gene recallOf all the genes that you entered, what proportion of them are associated with this keyphrase?
Matching genesThe number of genes that you entered that are associated with this keyphrase.
Total genesThe number of genes associated with this keyphrase in all of the literature that GADGET uses.
Back to top Back to GADGET Using the keyphrase search without a browser

You can use GADGET's keyphrase search without a browser by fetching http://gadget.biostat.wisc.edu/gadget/keywordlist?, and sending arguments via the querystring. The arguments that you can use are the same as the gene search.

Valid options for the orderby argument are total_abstracts, query_abstracts (matching abstracts), abstract_precision, abstract_recall, abstract_f1_score, total_genes, query_genes(matching genes), gene_precision, gene_recall, and gene_f1_score.

If you don't provide a limit argument, 5000 will be used by default.

Back to top Back to GADGET The interaction search

The GADGET interaction search allows you to browse through interactions describing interactions among genes. Interaction events are changes in states of biomolecules, and are automatically extracted from abstracts using natural language processing algorithms. Using the interaction search, you can learn about other genes that interact with a given gene, or gene interactions that are relevant to a keyword query.

The set of interactions we consider comes from the BioNLP '09 and BioNLP '11 Shared Task. The task defines an interaction event as "a change on the state of a bio-molecule or bio-molecules, e.g. phosphorylation of IkB involves a change on the protein IkB." The GADGET database only includes interactions that refer to more than one gene.

There are (currently) 9 different interaction types: gene expression, transcription, protein catabolism, localization, binding, phosphorylation, regulation, positive regulation, and negative regulation. Each interaction has one or more component "themes," which can be genes, or other interactions.

It is important to note that interaction extraction software is not perfectly accurate, so the set of interactions in the GADGET database is incomplete and contains some incorrect interactions. It is important to read each interaction's associated abstracts to learn about gene interactions.

GADGET uses interactions from the Turku NLP Group's EVEX dataset. Additional sets of interactions will be added in the future.

Back to top Back to GADGET Querying interactions

You can enter one or more gene names to display associated interactions. Term queries (entered in the same way as the gene search) first find abstracts that match the query, and then display interactions from those abstracts. Gene and keyphrase queries can be used together in combination.

For each interaction, GADGET will show you a diagram of the interaction, a list of genes referenced in the interaction, the interaction type (or types), and a list of abstracts that describe the interaction.

To narrow down the list of interactions, you can add genes to your query. Click on "See a list of genes referenced by these interactions..." to see which genes are referenced by interactions in your query, and how many interactions each gene is referenced by. Click on a gene to restrict your results to interactions that include that gene.

You can download interaction search results in XML or CSV formats. The XML representation contains more information about the structures of the interactions, but CSV's are often easier to use. You can download up to 500 interactions at a time. The XML representation uses a DTD file that can be found here.

Back to top Back to GADGET Searching interactions by gene symbol

GADGET allows you to supply a list of genes for interaction search queries. The interaction search doesn't currently support all of the features for gene entry that you can use in the other search modes.

When you provide a list of genes to GADGET's interaction search, GADGET will show you gene-interaction interactions that relate your input genes to each other and to other genes.

Genes should be entered using their Entrez Gene symbols. Use commas to separate multiple genes. Gene symbols are not case sensitive. GADGET will try to find human genes that match your gene symbols, and will show you an error message if it can't find one of them.

Back to top Back to GADGET Interaction diagrams

When you do an interaction search, GADGET displays diagrams of the interactions using a factor-graph-like representation. In the diagram, blue circles (nodes) represent genes, and lines between circles (edges) represent interactions.

For complex interactions, GADGET represents sub-interactions as purple circles. E.g. if an edge for a regulation interaction connects to a circle for a binding interaction, that interaction can be interpreted as regulation of binding.

Some interactions have a direction, such as regulation. The thickness of the lines in the diagrams represent the directionality of the interaction. The node connected the thin end of the line causes the node connected to the thick end of the line. Other interactions, such as binding, do not have a direction (the line has uniform thickness.)

Examples:

ACTR6 binds to RHO (not directional).
TP53 regulates CDKN1A (directional).
BMP2 positively regulates the phosphorylation of SMAD1
Back to top Back to GADGET Using the interaction search without a browser

You can access the interaction search without a browser by fetching http://gadget.biostat.wisc.edu/gadget/eventlist? and sending arguments via the query string at the end of the URL. To download data, you must include either &download=CSV or &download=XML in the query string, depending on which format you want. You can fetch up to 500 interactions at once.

To send information to GADGET, just append it to the url. Aguments are given by the name of the argument, an equals sign (=), and the argument's value, e.g. argument=value. Multiple arguments should be separated by ampersands (&). Here's a list of the arguments you can send:

downloadThis argument is required to download the results as a CSV or XML file. To download a CSV, use &download=CSV. To get XML, use &download=XML. The XML representation contains more information about the structure of the interactions, and uses a DTD file that can be found here.
qThe query terms of your search. You need to include either a query or a list of genes (or both).

In most browsers, you can include spaces and quotation marks in the URL. Special characters, like "&" and "/", must be percent encoded.
gene_entrez_idsA list of gene Entrez IDs separated by commas. GADGET will only return interactions which include all of these genes.

You must include either a query or a list of genes (or both). If both gene_entrez_ids and gene_symbols arguments are given, GADGET will ignore gene_symbols.
gene_symbolsA list of gene symbols separated by commas (see gene entry). GADGET will only return interactions which include all of these genes.

You must include either a query or a list of genes (or both). If both gene_entrez_ids and gene_symbols arguments are given, GADGET will ignore gene_symbols.
limitThe maximum number of interactions to return. This argument should be a number.

The maximum value for limit is 500. If no valid limit is specified, the results will be limited to 500 interactions. If you need to download more than 500 interactions, download multiple result sets using the offset argument.
offsetThe number of initial results to exclude from the result set. To omit the first 100 interactions, include &offset=100. If offset is omitted or invalid, GADGET will not omit any interactions.

If you do not provide at least one valid text query or gene list, or if no interactions match your criteria, the GADGET server will issue a 404 error.

http://gadget.biostat.wisc.edu/gadget/eventlist?download=XML&gene_symbols=SERPINC1&limit=10
Download a CSV of interactions in abstracts matching "limb regeneration":
http://gadget.biostat.wisc.edu/gadget/eventlist?download=CSV&q=limb regeneration
Download XML for interactions referencing BMP2 and SMAD7 (using Entrez IDs):
http://gadget.biostat.wisc.edu/gadget/eventlist?download=XML&gene_entrez_ids=650,4092
Download XML for interactions referencing SOX9 in abstracts matching "mesenchymal":
http://gadget.biostat.wisc.edu/gadget/eventlist?download=XML&gene_symbols=SOX9&q=mesenchymal
